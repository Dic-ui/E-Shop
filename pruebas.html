Introducción

En el presente trabajo se aborda el análisis y desarrollo de un sistema de gestión de incidencias orientado a mejorar la eficiencia operativa dentro de una organización. A través de la implementación de herramientas tecnológicas y metodologías ágiles, se busca optimizar el proceso de identificación, seguimiento y resolución de problemas técnicos o administrativos que puedan surgir en el entorno laboral.

Este proyecto se enmarca dentro del contexto académico de la asignatura de Ingeniería del Software, y ha sido elaborado por el Grupo 88 como parte de su formación práctica en el diseño y desarrollo de soluciones informáticas. Además, se presenta la evaluación de calidad del sistema E-Shop, una aplicación de comercio electrónico, aplicando el modelo ISO/IEC 25010 como guía metodológica.

Objetivos

  • Diseñar un sistema funcional que permita registrar y gestionar incidencias de manera estructurada y eficiente.
  • Aplicar principios de ingeniería del software para garantizar la calidad, escalabilidad y mantenibilidad del sistema propuesto.
  • Fomentar el trabajo colaborativo entre los integrantes del grupo, promoviendo la distribución equitativa de tareas y responsabilidades.
  • Desarrollar habilidades prácticas en el uso de tecnologías actuales como bases de datos, interfaces gráficas y control de versiones.
  • Evaluar el impacto del sistema en la mejora de procesos internos, mediante pruebas y validaciones que aseguren su correcto funcionamiento.

Modelo ISO/IEC 25010

El modelo ISO/IEC 25010 es un estándar internacional que define un marco para evaluar la calidad del software mediante ocho características principales:

  • Funcionalidad
  • Fiabilidad
  • Usabilidad
  • Eficiencia
  • Mantenibilidad
  • Portabilidad
  • Seguridad
  • Compatibilidad

En el caso de estudio E-Shop, estas dimensiones se aplicaron para asegurar que la plataforma de comercio electrónico cumpliera con los más altos estándares de calidad, garantizando estabilidad, seguridad y una experiencia de usuario óptima.

Plan de Pruebas

Para el sistema E-Shop, el plan de pruebas se diseñó siguiendo las etapas recomendadas por el modelo ISO/IEC 25010, priorizando las áreas más críticas del comercio electrónico: autenticación, carrito de compras y pagos.

Tipos de Pruebas

  • Pruebas funcionales: validación de flujos de registro, compra y pago.
  • Pruebas de rendimiento: simulación de hasta 500 usuarios concurrentes con Apache JMeter.
  • Pruebas de usabilidad: evaluación de accesibilidad y experiencia del usuario.
  • Pruebas de seguridad: detección de vulnerabilidades con OWASP ZAP y Burp Suite.
  • Pruebas de compatibilidad: ejecución en distintos navegadores y dispositivos móviles.
  • Pruebas de mantenibilidad: verificación de actualizaciones sin afectación de otros módulos.

Herramientas utilizadas

  • Selenium y JUnit para automatización funcional.
  • SonarQube para análisis de código.
  • Apache JMeter para rendimiento.
  • OWASP ZAP y Burp Suite para seguridad.
  • TestRail para la gestión de casos de prueba y métricas.

Resultados esperados

  • Tiempo de respuesta menor a 2 segundos.
  • Tasa de éxito en transacciones igual o superior al 99%.
  • Puntaje de satisfacción del usuario igual o superior a 8/10.
  • Sin vulnerabilidades críticas detectadas.

Conclusiones

El modelo ISO/IEC 25010 ha demostrado ser una herramienta eficaz para estructurar y ejecutar un proceso de evaluación de calidad integral, permitiendo identificar fortalezas y debilidades del software E-Shop en dimensiones clave como funcionalidad, seguridad y usabilidad.

La planificación y ejecución de pruebas en módulos críticos permitió validar el cumplimiento de requisitos funcionales y no funcionales, garantizando una experiencia de usuario satisfactoria y segura. El uso de herramientas especializadas facilitó la automatización, el análisis de rendimiento y la detección de vulnerabilidades, contribuyendo a una evaluación técnica más precisa y eficiente.

A pesar de las restricciones de tiempo y recursos, se logró una cobertura significativa de pruebas, cumpliendo con los criterios de salida establecidos y generando entregables valiosos para la toma de decisiones en el proyecto.
